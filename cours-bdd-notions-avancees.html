Bases de données avancées — MySQL

Du schéma correct au schéma performant : index, transactions, procédures et optimisation de requêtes.

Une fois le diagramme de classes acquis, on passe à l'implémentation optimisée. Cette partie transforme un schéma correct en schéma performant et sûr en production.

02 Normalisation (révision rapide)

  • 1FN — chaque colonne contient une valeur atomique (pas de liste, pas de valeurs multiples).
  • 2FN — pas de dépendance partielle à une partie seulement de la clé primaire composée.
  • 3FN — pas de dépendance transitive entre colonnes non-clés.
  • BCNF — version renforcée de la 3FN pour les cas de clés candidates multiples.
Exercice type Donner une table mal conçue (ex : Inscription(etudiant_nom, etudiant_email, cours_titre, enseignant_nom)) et faire identifier les anomalies de mise à jour avant de la décomposer.

03 Indexation

-- Index simple
CREATE INDEX idx_etudiant_email ON etudiant(email);

-- Index composite (ordre des colonnes = important)
CREATE INDEX idx_inscription_etudiant_cours ON inscription(etudiant_id, cours_id);

-- Voir le plan d'exécution
EXPLAIN ANALYZE
SELECT * FROM inscription WHERE etudiant_id = 12 AND cours_id = 4;

Point clé à enseigner : un index composite (a, b) sert aussi les requêtes filtrant sur a seul, mais pas celles filtrant sur b seul.

05 Procédures stockées, fonctions, triggers

DELIMITER //
CREATE TRIGGER maj_places_disponibles
AFTER INSERT ON inscription
FOR EACH ROW
BEGIN
  UPDATE cours SET places_restantes = places_restantes - 1
  WHERE id = NEW.cours_id;
END //
DELIMITER ;

DELIMITER //
CREATE PROCEDURE inscrire_etudiant(IN p_etudiant INT, IN p_cours INT)
BEGIN
  DECLARE places INT;
  SELECT places_restantes INTO places FROM cours WHERE id = p_cours;
  IF places > 0 THEN
    INSERT INTO inscription(etudiant_id, cours_id) VALUES (p_etudiant, p_cours);
  ELSE
    SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = 'Plus de places disponibles';
  END IF;
END //
DELIMITER ;

06 Vues

CREATE VIEW vue_moyennes_etudiants AS
SELECT e.id, e.nom, AVG(n.valeur) AS moyenne
FROM etudiant e
JOIN note n ON n.etudiant_id = e.id
GROUP BY e.id, e.nom;

07 Optimisation de requêtes

  • Éviter SELECT * — ne sélectionner que les colonnes nécessaires.
  • Utiliser EXPLAIN ANALYZE systématiquement sur une requête lente.
  • Préférer les JOIN explicites aux sous-requêtes corrélées quand c'est possible.
  • Activer et consulter le slow query log en développement.
-- Activer le slow query log
SET GLOBAL slow_query_log = 'ON';
SET GLOBAL long_query_time = 1;
