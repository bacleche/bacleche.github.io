Semaine 2 · Partie 1 — Conception UML

Diagrammes de séquence & d'activité

La dynamique du système : interactions entre objets et logique des processus métier.

Deux diagrammes complémentaires pour la dynamique du système : le diagramme de séquence pour les interactions objet-à-objet, le diagramme d'activité pour la logique des processus métier.

01 Diagramme de séquence

Pourquoi il est nécessaire

Il valide qu'un cas d'utilisation est réalisable techniquement et révèle les méthodes manquantes dans le diagramme de classes. On le construit à partir d'un seul scénario précis — jamais de l'ensemble du système d'un coup.

Éléments

ÉlémentNotationRôle
Ligne de vieLigne pointillée verticaleReprésente un objet dans le temps (haut → bas)
Message synchroneFlèche pleineAppel de méthode qui attend une réponse
Message asynchroneFlèche ouverteAppel sans attente de retour immédiat
RetourFlèche pointilléeRéponse renvoyée à l'appelant
ActivationRectangle fin sur la ligne de vieDurée pendant laquelle l'objet est actif

Exemple — PlantUML

@startuml
actor Etudiant
participant "Interface" as UI
participant "ServiceInscription" as SVC
database "BDD" as DB

Etudiant -> UI : demanderInscription(cours)
UI -> SVC : sInscrire(etudiant, cours)
SVC -> DB : verifierPlacesDisponibles(cours)
DB --> SVC : placesRestantes
SVC -> DB : creerInscription(etudiant, cours)
DB --> SVC : confirmation
SVC --> UI : inscriptionValidee
UI --> Etudiant : afficherConfirmation()
@enduml

02 Diagramme d'activité

Pourquoi il est nécessaire

Utile dès qu'un processus métier comporte des décisions, des boucles, ou des étapes parallèles (ex : workflow de validation d'une inscription avec vérification de prérequis).

Éléments

ÉlémentNotation
Nœud de début / finCercle plein / cercle bordé
ActionRectangle à coins arrondis
DécisionLosange
Fork / Join (parallélisme)Barre noire épaisse
Couloir (swimlane)Colonne verticale par acteur

Exemple — PlantUML

@startuml
start
:Étudiant soumet une demande d'inscription;
if (Prérequis validés ?) then (oui)
  if (Places disponibles ?) then (oui)
    :Créer l'inscription;
    :Envoyer confirmation;
  else (non)
    :Ajouter en liste d'attente;
  endif
else (non)
  :Rejeter la demande;
endif
stop
@enduml
Comment choisir entre les deux Séquence = "quels objets s'échangent quoi, dans quel ordre technique". Activité = "quelle est la logique métier avec ses conditions". Un même scénario peut avoir les deux, à des fins différentes.

03 Exercice pratique

À rendre en fin de séance

Diagramme de séquence : scénario "un enseignant saisit une note". Diagramme d'activité : processus "traitement d'une inscription" avec au moins une décision et un cas de rejet.
